The projects section of the client portal gives you full visibility into every security assessment associated with your account. From the dashboard summary to individual project timelines, you can track progress without waiting for a manual status update from your engagement manager.

## Dashboard summary

When you first sign in, the dashboard displays a summary row with four metrics:

| Metric | What it shows |
|---|---|
| **Total projects** | The total number of security assessment projects linked to your account |
| **Active projects** | Projects currently in progress or in the testing-complete phase |
| **Completed** | Projects that have been fully delivered and closed |
| **Reports available** | The count of delivered reports ready for you to access |

Below the summary row, the dashboard shows your five most recent projects so you can jump straight to active work.

## Browsing all projects

Select **Your Projects** on the dashboard to see a full list of every assessment linked to your account. Each project card shows:

- **Project name** and description
- **Service type** (e.g., web application, network, cloud)
- **Start date** and **end date**
- **Current status**

Click any project card or select **View Details** to open the full project detail view.

## Project detail view

The project detail view shows you everything about a single engagement:

- **Status** — the current phase of the project
- **Description** — a summary of the assessment objectives
- **Scope** — the specific systems, applications, or networks included in the assessment
- **Start date** and **end date** — the scheduled window for the engagement
- **Available reports** — reports that have been delivered for this project
- **Timeline** — a chronological log of milestones and updates

## Project status values

Each project carries one of three status values visible in the portal.

<AccordionGroup>
  <Accordion title="In progress">
    Active testing is underway. Our consultants are currently executing the assessment against the agreed scope. You can follow the timeline for updates as work progresses.
  </Accordion>
  <Accordion title="Testing complete">
    The hands-on testing phase has finished. Our team is now analyzing findings and authoring the final report. Expect a report to appear in the portal once it has been reviewed and delivered.
  </Accordion>
  <Accordion title="Completed">
    The engagement is fully closed. The final report has been delivered and is available in the portal. No further active work is taking place on this project.
  </Accordion>
</AccordionGroup>

## Project timeline

Every project has a timeline section that displays chronological updates and milestones posted by the engagement team. Each entry includes a title, an optional description, and the date it was recorded.

<Tip>
  Check the timeline regularly during active engagements. The team posts key milestones — such as reconnaissance complete, exploitation phase started, and report in review — so you always know where the assessment stands without needing to reach out directly.
</Tip>
